HIPAA Basics for IT Staff

Overview

The Health Insurance Portability and Accountability Act (HIPAA) sets national standards for protecting sensitive patient health information. As IT staff in a healthcare organization, you handle systems that store, process, and transmit Protected Health Information (PHI) daily. Understanding HIPAA's requirements isn't just compliance ‚Äî it's patient safety.

Key HIPAA Rules for IT

The Privacy Rule

Controls who can access PHI and how it's used. Key IT implications:

  • Minimum Necessary Standard: Only grant access to the minimum PHI needed for each role. Don't give everyone admin access "for convenience."
  • Access controls: Role-based access in every system. Document who has access to what and why.
  • Audit logs: Every system containing PHI must log who accessed what and when.

The Security Rule

Requires safeguards for electronic PHI (ePHI). Three categories:

  • Risk assessments (annual, document findings and remediation)
  • Incident response plan (documented, tested, updated)
  • Business Associate Agreements (BAAs) with all vendors handling PHI
  • Workforce sanctions for policy violations

Technical Safeguards
  • Access control: Unique user IDs, role-based permissions, automatic logoff
  • Encryption: ePHI encrypted at rest (AES-256) and in transit (TLS 1.2+)
  • Audit controls: Logging of all access to systems containing ePHI
  • Integrity controls: Mechanisms to prevent unauthorized alteration of ePHI
  • Transmission security: VPN for remote access, encrypted email for PHI

  • Server room access restricted (badge access, logs)
  • Workstation security (screen locks, privacy screens in clinical areas)
  • Device disposal (NIST 800-88 compliant wiping before recycling/disposal)
  • Asset tracking (Snipe-IT ‚Äî know where every device with ePHI access is)

The Breach Notification Rule

When a breach occurs, specific notification timelines kick in:

  • Individual notification: Within 60 days of discovery
  • HHS notification: Within 60 days (if 500+ individuals affected, immediate media notice too)
  • IT's role: Detect, contain, investigate, and document the breach. Provide technical findings for the breach assessment.

IT Daily Operations: HIPAA Checklist

User Account Management

  • ‚òê New accounts: Minimum necessary access. Document role-based justification.
  • ‚òê Terminated employees: Disable accounts within 4 hours of termination notice.
  • ‚òê Transfers: Review and adjust access when employees change roles.
  • ‚òê Quarterly access reviews: Audit who has access to ePHI systems. Remove unnecessary access.
  • ‚òê Password policies: 12+ characters, 90-day rotation, MFA on all ePHI systems.

System Security

  • ‚òê Patch management: Apply critical security patches within 72 hours.
  • ‚òê Antivirus/EDR: Ensure all endpoints have current definitions.
  • ‚òê Encryption: Verify BitLocker (Windows) or FileVault (Mac) is enabled on all devices.
  • ‚òê Backups: Test backup restoration quarterly. Encrypt backup media.
  • ‚òê Firewall rules: Review and document. Remove unnecessary open ports.

Vendor Management

  • ‚òê BAAs: Every vendor that touches PHI must have a signed Business Associate Agreement.
  • ‚òê Cloud services: Verify HIPAA compliance (Google Workspace BAA, AWS BAA, etc.).
  • ‚òê Shadow IT: Watch for employees using unauthorized cloud services for PHI.

Common HIPAA Mistakes in IT

  • ‚ùå Sharing login credentials: Every user must have a unique ID. "Shared" accounts eliminate audit trails.
  • ‚ùå Sending PHI via unencrypted email: Use the organization's encrypted email solution or secure file sharing.
  • ‚ùå Leaving screens unlocked: Enforce automatic screen lock (5 minutes max for workstations accessing ePHI).
  • ‚ùå No BAA with cloud vendors: Using Dropbox, personal Gmail, or other consumer services for PHI is a violation ‚Äî even if "it's just one file."
  • ‚ùå Not logging access: If a system contains ePHI and doesn't have audit logging enabled, it's a compliance gap.
  • ‚ùå Delayed account deactivation: A terminated employee accessing systems is both a security risk and a HIPAA violation.
  • ‚ùå Improper device disposal: Throwing away a hard drive without proper wiping can result in a reportable breach.

Incident Response: What IT Does During a Breach

1
Contain

Stop the breach. Disable compromised accounts, isolate affected systems, block malicious IPs. Document every action with timestamps.

2
Investigate

Determine what happened, what data was affected, and how many individuals are impacted. Review logs, conduct forensics.

3
Assess

Apply the four-factor breach assessment: (1) Nature of PHI involved, (2) Who accessed it, (3) Was PHI actually acquired/viewed, (4) Extent of risk mitigation.

4
Notify

Work with compliance and legal to determine notification requirements. Provide technical documentation for the breach report.

5
Remediate

Fix the vulnerability that caused the breach. Update policies, retrain staff, implement additional safeguards. Document everything.

Quick Reference: PHI Identifiers

HIPAA defines 18 identifiers that make health information "protected." IT systems must safeguard any data containing these:

  1. Name
  2. Geographic data (address, ZIP code smaller than state)
  3. Dates (birth, admission, discharge, death)
  4. Phone numbers
  5. Fax numbers
  6. Email addresses
  7. Social Security numbers
  8. Medical record numbers
  9. Health plan beneficiary numbers
  10. Account numbers
  11. Certificate/license numbers
  12. Vehicle identifiers (VIN, plates)
  13. Device identifiers (serial numbers)
  14. Web URLs
  15. IP addresses
  16. Biometric identifiers
  17. Full-face photographs
  18. Any other unique identifying number
